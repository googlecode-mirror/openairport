Quick Info

All airports can benefit from the OpenAirport.org system. From issuing graphic field condition reports to conducting airfield safety self inspections the OpenAirport.org system can be everything you want it do be!

OpenAirport.org - Examples For You

For every aspect of an airports Part 139 needs there is a module programmed or in development that meets or commonly exceeds the needs of your certification inspector. Listed below are some of the forms most closely related to Part 139 and their associated requirements. The OpenAirport.org system can do these forms as well as many others related to an airports continuing operation.


Airport Duty Log
Screens Description

The Airport Duty Log tracks everyones movement in the system. It displayes when a user logs in, logs out, prints a report, displays a report, or enters a new report. By keeping track of when and who does something you can ensure airport tenants are kept informed on the latest airport information.

Part 139.303 Personnel
Screens Description

The user starts by selecting the type of training session that was conducted, in this case a Operations Personnel training session was selected.

After checking the topices that were taught the user clicks the submit button and then can attach the training attendance sheet and outline.

The user may at any time browse the training history of each employee at the airport as well as any of the classes which have been held.

Part 139.327 Airport Safety Self Inspections (Video Demo)
Screens Description

When a user starts a new Part 139.327 inspection they first select the type of inspection. Be it Regularly Scheduled, Periodic Inspection, or whatever.

After selecting the type of inspection, corresponding checklist items are displayed. The checklist items displayed are dependent on the type of inspection selected.

There may be an unlimited number of checklist items.

After going through the steps to create the inspection you end up with this inspection report.

Every discrepancy noted will be reported on the sheet in view . Because the system is closed-looped, maintenance can then fix the discrepancy and a record is created for operations to check the discrepancy.

Part 139.337 Wildlife Hazard Management Reports
Screens Description

After conducting a Part 139.337 action, the user would start the process very much like every other report. They enter all of the applicable information into the form. The user should take care to add as much information about the event as possible in the results field and click on an airport map where the incident took place.

With the report created the user has the ability to browse all history of events and even create a chart which displayes every incident on it from a selected time period. Moving the mouse over a target will display an informational window about the incident.

Part 139.339 Field Condition Reports
Screens Description

When a user starts a Part 139.339 Field Condition Report, they start the process very much like the 327 report. First they select the type of Field Condition Report they will be conducting. This allows for users to conduct field conditions of only certain parts of the field or field condition forms for differen periods of the year, or even after major contruction at the airport has added or removed portions of the airfield.

After selecting the type of Field Condition Report, the appliable checklist items are displayed. Once again the checklist is dynmiac and depends on the type of condition report selected. To the left is an example.

There may be an unlimited number of checklist items.

After going through the steps to create the field condition report, the user can print out a report that looks like the one to the left.

If the user chose to add special advisories to the field condition report, they will also be displayed in a format a little different than the Pat 139.327 discrepancies, but will convay similar types of information. The user can even draw lines on the map to select an odd shaped area, or do whatever they wish with the special advisory function.

Part 139.339.b Notice to AirMen
Screens Description

The Notice to AirMen (NOTAM) and Field Condition Report both function in the same way. The user selects the type of Field Condition Report to use as the base for the airport surfaces checked, in the example provided it is a new time period after a new taxiway was added to the airport.

The user then clicks the surfaces that are closed and enteres a decription in the field, and enters all other applicale information such as the person he talked to at Lockheed, the airline, FBO, etc.

After clicking the submit button the user can even add a graphic to the report to show where an area not directly related to a surface is closed. It is truly a very graphic NOTAM avilable to anyone.
